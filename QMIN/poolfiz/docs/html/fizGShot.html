ShotStatus execute(fizGTable& tbl, fizGTableState& state, double a, double b, double theta, double phi, double V, bool verbose = false)
Executes a shot on the grapical GTable. Returns a ShotStatus enum indicating the result.
Parameters:
tab - reference to the fizGTable on which to execute the shot
state - reference to the fizGTableState with which to execute the shot
a - right/left english in mm
b - top/bottom english in mm
theta - cue stick elevation in degrees
phi - cue stick azimuth angle (heading) in degrees
V - pre-impact cue stick velocity in m/s (max. 4.5 m/s)
verbose - set to true for console output of shot data (optional, default = false)

vector<fizEvent> getEventList()
Returns the time-sorted list of events that occured on the last shot, in the form of a Standard Template Library vector (array) of fizEvent objects.

bool getStateAt(double time, fizGTableState &state)
Updates the provided fizGTableState to reflect the state at the given absolute time. Returns true if the provided time is less than the time of the last event that occurred during the last shot. This is used for shot animation in examples 1 and 3
Parameters:
time - absolute time in seconds (every shot begins at t=0 by definition)
state - reference to the fizGTableState

double getLastTime()
Returns the time of the last event in the list.

bool save(string tag = "", bool autogen = true)
Saves a text representation of the shot to the working directory. If autogen is not supplied or is supplied and is true, the filename is automatically generated and contains the optional string tag. Otherwise if autogen is false the string tag itself is used for the filename.

bool load(string fname, fizGTableState& state, double &a, double &b, double &theta, double &phi, double &V)
Loads a shot from the specified file.
Parameters:
fname - the filename, including extension, of the shot to load
state - reference to the initial table state to be populated from the file
a - reference to the right/left english to be populated from the file
b - reference to the top/bottom english to be populated from the file
theta - reference to the cue stick elevation to be populated from the file
phi - reference to the cue stick azimuth angle (heading) to be populated from the file
V - reference to the pre-impact cue stick velocity to be populated from the file

bool saveDynamics(double timestep, string tag = "")
Saves a verbose dump of the dynamics of all balls with the given timestep resolution. The filename is automatically generated but contains the optional string tag.
Parameters:
timestep - the timestep in seconds for the dynamics. The file size is inversely proportional to the timestep.
tag - the optional tag string for the filename.
