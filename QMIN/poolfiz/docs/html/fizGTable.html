Documentation

A billiard table for use with graphical output. You should only ever need one instance of a fizGTable, but we have provided a copy constructor or assignment operator.
fizGTable()
Constructs a table with the default parameters

fizGTable( bool onRobot )
Constructs a table with the given parameters and automatically sets the pockets.
Parameters:
- onRobot indicates whether the game is being played on the robotic table
Since:
v2.5.0

fizGTable( double length, double width, double cornerPocketWidth, double sidePocketWidth, double muS, double muR, double muSp, double railHeight, double cueLength, double railVelDamping, double railSpinDamping, double railZSpinAngleAdj, double railVelAngleAdj, bool onRobot = false )
Constructs a table with the given parameters and automatically sets the pockets.
Parameters:
length - the length of the table in metres
width - the width of the table in metres
cornerPocketWidth - the width of corner pockets (horn to horn) in metres
sidePocketWidth - the width of the side pockets (horn to horn) in metres
muS - the coefficient of sliding friction (dimensionless)
muR - the coefficient of rolling friction (dimensionless)
muSp - the coefficient of spinning friction (dimensionless)
railHeight - the height of the top of the rail above the table in metres
cueLength - the length of the cue in metres
railVelDamping - velocity damping factor of the banks
railSpinDamping - spin damping factor of the banks
railZSpinAngleAdj - z-spin angle of deflection factor of the banks
double - railVelAngleAdj velocity deflection factor of the banks
- onRobot indicates whether the game is being played on the robotic table (optional, default = false)
Since:
v2.5.0

int isValidBallPlacement( fizGTableState& state )
Returns BAD_X_VAL or BAD_Y_VAL if any ball that is in play is off the table or BALL_OVERLAP if any ball is overlapping with another ball. Used to check cue ball placement validity in the case of Ball In Hand, for example.
Parameters:
state - the table state to check
Since:
v2.2.0

int isPhysicallyPossible(fizGTableState& state, double a, double b, double theta, double phi, double V)
Checks physical validity of the given shot parameters and table state. Used before calling fizShot::execute() to double-check that an error condition will not result due to invalid parameters.
